# UnityRpg — РПГ-симуляция боя (быстрый старт)

Чистая симуляция в консоль Unity: хост владеет юнитами и «физикой» боя,
скрипты режиссируют — комментатор, ярость босса, подкрепление.

## Размещение файлов

```
Assets/
  Dsl.Core/                                  ← ядро (из репозитория)
  Dsl.Unity/                                 ← адаптер (из репозитория)
  Examples/UnityRpg/RpgSimulation.cs         ← ХОСТ: регистрация API + симуляция
  StreamingAssets/
    Scripts/
      rpg_demo/                              ← СКРИПТЫ (папка StreamingAssets
        module.json                             отсюда копируется в Assets целиком)
        src/
          battle.script
```

## Шаги

1. Установите пакет `com.unity.nuget.newtonsoft-json` (Package Manager → Add by name).
2. Скопируйте `Dsl.Core` и `Dsl.Unity` в `Assets/`.
3. Скопируйте `RpgSimulation.cs` в любое место `Assets/`.
4. Скопируйте папку `StreamingAssets` из этого примера в `Assets/`
   (получится `Assets/StreamingAssets/Scripts/rpg_demo/...`).
5. Пустая сцена → пустой GameObject → добавьте компонент **RpgSimulation** → Play.

## Поиграться

- Правьте `battle.script` прямо во время Play — сохранение перекомпилирует
  модуль на лету; при ошибке в консоли будет файл:строка, а бой продолжит
  идти на старой версии скрипта.
- Порог ярости (`0.4`), сила регена, размер подкрепления — всё в скрипте.
- Баланс юнитов (hp/урон/кулдауны) — в `StartBattle()` на стороне хоста.
- Для сетевой игры (FishNet) наследуйтесь и переопределите
  `IsAuthority => netObject.IsServerInitialized` — скрипты будут жить только
  на сервере.
