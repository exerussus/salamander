# Salamander — расширение VS Code

Подсветка синтаксиса, ошибки компиляции и автодополнение для `.sal`.

## Установка (dev)

```bash
cd Tools/vscode-salamander
npm install
npm run compile
```

Дальше два варианта:
- **Отладка:** открыть эту папку в VS Code → F5 (Extension Development Host);
- **Постоянная установка:** `npx @vscode/vsce package` → в VS Code
  «Extensions → … → Install from VSIX».

## Ошибки компиляции (нужен CLI-чекер)

```bash
cd Tools/DslCheck
dotnet build
```

В настройках VS Code укажите:
- `salamander.checkerPath` → `<repo>/Tools/DslCheck/bin/Debug/net8.0/DslCheck.dll`

Проверка запускается при сохранении `.sal`/`module.json` и командой
«Salamander: проверить скрипты». Диагностики подсвечиваются в файлах
и попадают в панель Problems.

## Автодополнение API игры

Расширение читает `salamander-api.json` — манифест всего, что игра открыла
скриптам (енумы, классы со свойствами, API-методы, события). Игра выгружает
его автоматически: запустите сцену в редакторе Unity один раз — файл появится
рядом с модулями (`StreamingAssets/Scripts/salamander-api.json`). Расширение
найдёт его само (или укажите путь в `salamander.apiManifest`).

Что умеет дополнение:
- `Engine.` → встроенные методы движка с сигнатурами;
- `UnitApi.` / любой API-класс → его методы со сниппетами аргументов;
- `Team.` / любой енум → элементы;
- `event ` внутри триггера → полные сигнатуры хостовых событий (вставляется
  готовый обработчик с параметрами);
- имена триггеров/классов/енумов/функций из `.sal`-файлов воркспейса;
- hover по методам API и событиям — сигнатура.

Без `salamander-api.json` работают подсветка, сниппеты и символы воркспейса.
